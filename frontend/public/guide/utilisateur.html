Journee type par profil

Journee type — Exploitation (Sophie GIRARD)

Matin :
  1. Consulter le Planning de la semaine — verifier que tous les conducteurs ont des missions
  2. Ouvrir Tournees → generer les executions de la semaine si pas encore fait
  3. Ouvrir Executions → dispatcher les tournees du jour (bouton « Dispatcher »)
  4. Verifier les Notifications (cloche) — alertes de conformite, conflits de planning
Journee :
  1. Surveiller les missions En cours dans la liste des missions
  2. Traiter les missions manuelles : creer, planifier, affecter conducteur + vehicule
  3. Traiter les Litiges ouverts — repondre, documenter, cloturer
  4. Gerer les offres d'Affretement si des missions sont sous-traitees
Fin de journee :
  1. Verifier les missions Livrees — valider les POD (preuves de livraison)
  2. Cloturer les missions terminees (Livree → Cloturee)
  3. Preparer le planning du lendemain

Journee type — Comptabilite (Claire MOREAU)

Matin :
  1. Consulter le Tableau de bord — DSO, balance agee, factures impayees
  2. Traiter les Relances en cours — envoyer les rappels niveau 2/3
  3. Verifier les Notifications — factures arrivant a echeance
Journee :
  1. Creer les Factures pour les missions cloturees
  2. Numiriser les factures fournisseurs via OCR
  3. Rapprocher les Factures fournisseurs — verifier les ecarts
  4. Mettre a jour les Regles de tarification si besoin
Fin de mois :
  1. Valider toutes les factures du mois
  2. Generer les avoirs si necessaire
  3. Exporter les donnees pour la comptabilite

Journee type — RH / Paie (Isabelle FOURNIER)

Matin :
  1. Consulter la Conformite des conducteurs — traiter les alertes rouges en priorite
  2. Verifier les Notifications — documents expirant, conducteurs desactives
  3. Deposer les documents renouveles (permis, FIMO, visite medicale)
Journee :
  1. Mettre a jour les fiches Conducteurs — entrees, sorties, changements de contrat
  2. Acquitter les Alertes de conformite traitees
Fin de mois :
  1. Creer la Periode de paie du mois
  2. Cliquer sur « Calculer » — le systeme genere les variables depuis les missions
  3. Verifier et corriger les variables (heures, km, primes)
  4. Soumettre → Approuver → Verrouiller
  5. Exporter SILAE — envoyer au cabinet comptable

Journee type — Flotte (Thomas ROUX)

Matin :
  1. Consulter le Tableau de bord flotte — disponibilite, maintenances a venir
  2. Verifier la Conformite vehicules — documents expires (CT, assurance, tachygraphe)
  3. Traiter les Notifications — alertes d'expiration
Journee :
  1. Planifier les Maintenances — ajouter les RDV, renseigner les prestataires
  2. Enregistrer les maintenances effectuees (couts, pieces, resultat)
  3. Saisir les Couts vehicules (carburant, peages, assurance)
  4. Traiter les Sinistres en cours — photos, devis, suivi assurance
Hebdomadaire :
  1. Deposer les documents vehicules renouveles dans la Conformite
  2. Mettre a jour les fiches vehicules (km compteur, changement de statut)

Journee type — Administrateur (Marc LEFEVRE)

Matin :
  1. Consulter le Tableau de bord complet — CA, marge, conformite, DSO
  2. Verifier les Notifications — tous les evenements critiques
Hebdomadaire :
  1. Gerer les Utilisateurs — creer de nouveaux comptes, ajuster les roles
  2. Configurer les Regles de notification dans Parametres
  3. Consulter l'Audit — verifier les actions effectuees
  4. Mettre a jour les Parametres entreprise si besoin (RIB, TVA, centres de cout)

Procedures quotidiennes

A. Creer une mission manuelle

  1. Aller dans Exploitation → Missions
  2. Cliquer sur « Nouvelle mission »
  3. Renseigner : client, type, dates de chargement/livraison, montant
  4. La mission est creee en statut Brouillon

B. Creer une tournee recurrente

  1. Aller dans Exploitation → Tournees
  2. Cliquer sur « Nouveau modele »
  3. Renseigner : code, libelle, client, recurrence (LUN_VEN, quotidienne, etc.)
  4. Definir les valeurs par defaut : conducteur, vehicule, montant vente
  5. Ajouter les arrets (points de livraison) dans l'onglet Arrets

C. Generer les executions de la semaine

  1. Ouvrir le detail d'un modele de tournee
  2. Aller dans l'onglet Executions
  3. Renseigner la plage de dates (ex: lundi → vendredi)
  4. Cocher « Creer les missions automatiquement »
  5. Cliquer sur « Generer »
  6. Le systeme cree une execution par jour ouvrable + une mission par execution
Idempotence : Si vous regenerez pour les memes dates, les executions deja creees ne seront pas dupliquees.

D. Affecter un conducteur et un vehicule

  1. Ouvrir la fiche d'une mission
  2. Dans la carte Affectation, selectionner le conducteur et le vehicule
  3. Si sous-traite, cocher « Sous-traite » et renseigner le sous-traitant + montant achat
  4. Cliquer sur « Affecter »
  5. Le statut passe a Affectee
Attention : Le systeme verifie que le conducteur est actif, que le vehicule est actif, et signale les conflits de planning (non bloquant).

E. Suivre le cycle de vie d'une mission

Brouillon → Planifiee → Affectee → En cours → Livree → Cloturee → Facturee
  • Brouillon → Planifiee : quand les informations sont completes
  • Planifiee → Affectee : quand conducteur + vehicule sont assignes
  • Affectee → En cours : quand le conducteur demarre la livraison
  • En cours → Livree : quand tous les points de livraison sont livres
  • Livree → Cloturee : quand le POD (preuve de livraison) est valide
  • Cloturee → Facturee : quand la facture est emise

F. Consulter le planning

  1. Aller dans Exploitation → Planning
  2. Basculer entre les onglets Conducteurs et Vehicules
  3. Naviguer par semaine avec les fleches
  4. Cliquer sur une mission (pastille coloree) pour voir son detail

G. Gerer un litige

  1. Ouvrir une mission → onglet Litiges
  2. Cliquer sur « Nouveau litige »
  3. Renseigner : type (avarie, perte, retard…), description, responsabilite, montant estime
  4. Le litige est cree en statut Ouvert

J. Piloter une execution (tournee du jour)

Une fois les executions generees, voici le cycle de vie complet :

Brouillon → Planifiee → Dispatchee → En cours → Terminee
  1. Generer : Depuis le modele, onglet Executions → choisir les dates → « Generer »
  2. Verifier l'affectation : Ouvrir l'execution, verifier conducteur et vehicule
  3. Affecter des missions : Onglet Missions → « Affecter une mission » → saisir l'ID
  4. Reordonner : Les missions ont un numero de sequence dans l'execution
  5. Dispatcher : Cliquer sur « Dispatcher » pour envoyer au conducteur
  6. Demarrer : Cliquer sur « Demarrer » quand le conducteur part
  7. Terminer : Cliquer sur « Terminer » — le systeme calcule automatiquement les totaux financiers
Retirer une mission : Dans l'onglet Missions de l'execution, cliquer sur « Retirer » pour desassocier une mission sans la supprimer.

Procedures quotidiennes

A. Creer une facture client

  1. Aller dans Finance → Factures
  2. Cliquer sur « Nouvelle facture »
  3. Selectionner le client et la periode
  4. Ajouter les lignes (missions cloturees ou lignes manuelles)
  5. Verifier le total HT, TVA, TTC
  6. Cliquer sur « Valider » pour finaliser la facture
  7. Telecharger le PDF si besoin

B. Gerer les regles de tarification

  1. Aller dans Finance → Tarification
  2. Creer des regles : forfait, prix au km, surcharge
  3. Associer a un client specifique ou laisser en global
  4. Le systeme appliquera ces regles lors de la generation de factures

C. Suivre les impayes (relances)

  1. Aller dans Finance → Relances
  2. Les factures en retard sont listees par niveau de relance (1, 2, 3)
  3. Pour chaque facture, vous pouvez enregistrer une action de relance
  4. Suivre l'historique des relances effectuees

D. Numeriser une facture fournisseur

  1. Aller dans Finance → OCR
  2. Telecharger (upload) le scan de la facture (PDF ou image)
  3. Le systeme extrait automatiquement : numero, montant, date, fournisseur
  4. Verifier et corriger les champs extraits
  5. Valider pour creer la facture fournisseur

E. Rapprocher les factures fournisseurs

  1. Aller dans Finance → Factures fournisseurs
  2. Pour chaque facture, verifier l'ecart entre le montant facture et le montant attendu (missions)
  3. Valider ou contester la facture

KPIs a surveiller

  • DSO : Delai moyen de recouvrement (objectif < 45 jours)
  • Balance agee : Repartition des impayes par anciennete
  • Nombre de factures impayees
  • Ecarts sous-traitants : Ecart entre couts prevus et factures recues

4. Profil RH / Paie

Procedures quotidiennes

A. Creer un conducteur

  1. Aller dans Referentiels → Conducteurs
  2. Cliquer sur « Nouveau conducteur »
  3. Renseigner : nom, prenom, matricule, permis, qualifications (FIMO, FCO, ADR)
  4. Renseigner le type de contrat (CDI, CDD, Interim)
  5. Le conducteur est cree en statut Actif

B. Suivre la conformite des conducteurs

  1. Aller dans Referentiels → Conformite
  2. Les conducteurs avec des documents manquants ou expires apparaissent en rouge (Bloquant) ou orange (A regulariser)
  3. Cliquer sur un conducteur pour voir la liste des documents requis
  4. Telecharger les documents manquants (permis, FIMO, visite medicale, etc.)
Conformite bloquante : Un conducteur en statut « Bloquant » ne peut pas etre affecte a une mission. Regularisez ses documents en priorite.

C. Preparer la paie mensuelle

  1. Aller dans Finance → Paie
  2. Creer une nouvelle periode de paie (mois/annee)
  3. Cliquer sur « Calculer » pour generer les variables depuis les missions du mois
  4. Le systeme calcule pour chaque conducteur : heures, km, missions completees
  5. Corriger les variables si necessaire
  6. Cliquer sur « Soumettre » pour envoyer a la validation
  7. Apres approbation, « Verrouiller » et « Exporter SILAE »
Cycle de la paie : Brouillon → Soumise → Approuvee → Verrouillee

KPIs a surveiller

  • Delai de pre-paie : Nombre de jours entre fin de mois et export SILAE
  • Anomalies : Variables de paie hors normes
  • Conformite conducteurs : % de conducteurs avec tous les documents valides

Procedures quotidiennes

A. Ajouter un vehicule

  1. Aller dans Referentiels → Vehicules
  2. Cliquer sur « Nouveau vehicule »
  3. Renseigner : immatriculation, marque, modele, categorie, PTAC, charge utile
  4. Renseigner : proprietaire (propre/location), assurance, controle technique

B. Planifier une maintenance

  1. Aller dans Flotte → Maintenance
  2. Cliquer sur « Planifier »
  3. Selectionner le vehicule, le type de maintenance, la date prevue
  4. Renseigner le prestataire et le cout estime
  5. La maintenance apparait dans le calendrier

C. Enregistrer un sinistre

  1. Aller dans Flotte → Sinistres
  2. Cliquer sur « Nouveau sinistre »
  3. Renseigner : date, type (collision, vol, vandalisme), description
  4. Renseigner le conducteur implique, le tiers eventuel
  5. Joindre les photos et documents
  6. Suivre le sinistre jusqu'a sa cloture

D. Suivre la conformite vehicules

  1. Aller dans Referentiels → Conformite
  2. Les vehicules avec des documents expires apparaissent en rouge
  3. Documents a surveiller : carte grise, assurance, controle technique, limiteur de vitesse, tachygraphe
  4. Telecharger les documents mis a jour

E. Suivre les couts

  1. Aller dans Flotte → Tableau de bord
  2. Consulter le cout mensuel par categorie (carburant, assurance, maintenance, peages)
  3. Identifier les vehicules les plus couteux

KPIs a surveiller

  • Taux de disponibilite : % de vehicules actifs (objectif > 90%)
  • Cout au km : Cout total / km parcourus
  • Maintenances a venir : Nombre de maintenances prevues dans les 30 prochains jours
  • Pannes non planifiees : Nombre de pannes/immobilisations imprevues

Procedures cles

  1. Aller dans Parametrage → Parametres

  3. Ajouter les comptes bancaires (IBAN, BIC)
  4. Configurer les taux de TVA applicables
  5. Creer les centres de cout

B. Gerer les utilisateurs

  1. Aller dans Administration → Entreprises → selectionner votre entreprise
  2. Dans l'onglet Utilisateurs, cliquer sur « Nouvel utilisateur »
  3. Renseigner : email, nom complet, role, agence
  4. Chaque utilisateur recoit les menus et permissions de son role

C. Consulter l'audit

  1. Aller dans Parametrage → Audit
  2. Filtrer par utilisateur, action, type d'entite ou date
  3. Voir le detail : ancienne valeur → nouvelle valeur

KPIs a surveiller

  • CA mensuel : Chiffre d'affaires du mois
  • Marge brute : Ventes - Achats
  • Taux de conformite : % de conducteurs/vehicules conformes
  • DSO : Delai de recouvrement
  • Cout au km : Cout total flotte / km
  • Missions en cours : Volume d'activite en temps reel
  • Litiges ouverts : Nombre de reclamations non resolues

Procedures quotidiennes

  1. Ouvrir une mission

9. Conformite et documents reglementaires

📋
Gestion de la conformite
Suivi des documents obligatoires pour conducteurs, vehicules et sous-traitants
Qui gere la conformite ?
  • RH / Paie : documents des conducteurs (permis, FIMO, FCO, ADR, visite medicale)
  • Flotte : documents des vehicules (carte grise, assurance, controle technique, tachygraphe)
  • Administrateur : tous les documents + configuration des modeles

A. Tableau de bord conformite

  1. Aller dans Referentiels → Conformite
  2. La page affiche un resume global :
IndicateurDescription
Entites totalNombre total de conducteurs + vehicules + sous-traitants
ConformesEntites avec tous les documents valides (vert)
A regulariserEntites avec des documents manquants ou expirant bientot (orange)
BloquantsEntites avec des documents obligatoires manquants/expires (rouge)
Taux de conformiteBarre de progression en pourcentage (vert ≥ 80%, orange ≥ 50%, rouge < 50%)

Filtrez par type d'entite avec les onglets : Tous, Conducteurs, Vehicules, Sous-traitants.

Le tableau liste chaque entite avec ses indicateurs : documents requis, valides, manquants, expires.

B. Consulter et deposer les documents d'une entite

  1. Dans le tableau de conformite, cliquer sur « Voir » a cote d'un conducteur ou vehicule
  2. Vous arrivez sur la checklist de conformite de l'entite
  3. Pour chaque type de document requis, le statut est affiche :
IconeStatutSignification
✔OKDocument present et valide
⏰ExpirantDocument bientot expire (dans les prochains jours)
❌ExpireDocument expire — a renouveler d'urgence
❓ManquantDocument requis non encore depose
⏳En attenteDocument depose, en attente de validation

Deposer un document

  1. Dans la checklist, cliquer sur le type de document a deposer
  2. Renseigner les champs :
ChampObligatoireDescription
FichierOuiScan PDF ou photo du document
Date d'emissionNonDate de delivrance du document
Date d'expirationNonDate de fin de validite
Numero de documentNonNumero du permis, de la carte, etc.
Organisme emetteurNonQui a delivre le document

C. Documents types par entite

Conducteurs — documents suivis

  • Permis de conduire (categories B, C, CE)
  • Carte d'identite / passeport
  • FIMO (Formation Initiale Minimum Obligatoire)
  • FCO (Formation Continue Obligatoire)
  • ADR (matieres dangereuses, si applicable)
  • Visite medicale du travail
  • Carte conducteur (tachygraphe)
  • Attestation de capacite professionnelle

Vehicules — documents suivis

  • Carte grise (certificat d'immatriculation)
  • Attestation d'assurance
  • Controle technique
  • Limiteur de vitesse (certificat de verification)
  • Tachygraphe (certificat de verification)
  • ATP (attestation pour transport de denrees perissables, si applicable)
  • ADR (conformite matieres dangereuses, si applicable)

D. Banniere d'alerte sur les fiches

Quand vous ouvrez la fiche d'un conducteur ou d'un vehicule :

  • Si des documents sont expires : une banniere rouge apparait en haut de la page
  • Si des documents expirent bientot : une banniere orange apparait
  • Dans les listes de conducteurs/vehicules, les lignes sont surlignees selon le statut de conformite
Impact operationnel : Un conducteur ou vehicule en statut « Bloquant » (rouge) ne peut pas etre affecte a une mission. L'exploitation sera alertee si elle tente l'affectation.

E. Alertes de conformite

  1. Aller dans Referentiels → Conformite → bouton « Alertes »
  2. La page liste toutes les alertes actives :
Type d'alerteDeclenchement

Filtrez les alertes par statut : Toutes, Envoyees, Acquittees, Escaladees.

Pour traiter une alerte, cliquer sur « Acquitter » et optionnellement ajouter une note (ex: « Document commande, reception prevue le 15/04 »).

F. Modeles de conformite (administrateur)

  1. Aller dans Referentiels → Conformite → bouton « Modeles »
  2. Les modeles definissent quels documents sont obligatoires par type d'entite
  3. Pour chaque document type, configurer :
  • Type d'entite : conducteur, vehicule, ou sous-traitant
  • Obligatoire / optionnel
  • Bloquant : si manquant, empeche l'affectation aux missions
  • Duree de validite par defaut
  • Seuil d'alerte (en jours avant expiration)
